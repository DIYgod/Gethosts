常见问题

Hosts文件
想要下载Chrome吧提供的hosts文件请点这里，解压后移动到C:\Windows\System32\drivers\etc（如果已存在请覆盖，如果想恢复只需直接删除该文件，不会对系统造成影响）。完成后在命令行程序中执行ipconfig /flushdns即可即时生效。
linux系统下，在解压好的文件所在位置执行sudo cp ./hosts /etc/hosts && sudo /etc/init.d/networking restart即可

啥，有解压密码？快回去原帖三楼查找。Mac用户请用第三方解压软件解压，系统暂时无法解压（不知道出了什么问题）
如何还原修改状态
如果想恢复只需直接删除该文件，不会对系统造成影响
改了hosts文件依然没有效果
目前有N种可能
1.新疆地区屏蔽了Google 北京 IP，请重新下载一份新疆专用的hosts
非新疆用户最好不要使用以避免下载扩展时出现Download interrupted
2.你的网络与GGC IP连接的443端口被屏蔽掉了
按照这里的提示测试能否连通hosts内置IP
3.hosts已被和谐，尝试更换压缩文件内“必看”提供的备用IP或等待更新
4.你的 “安全软件” 很 “负责任”的把修改完的hosts改了回去，添加白名单并重新覆盖
5.某些网站需要 https 才可以打开，例如Google快照并没有强制https访问
下载扩展出现Download interrupted
1.hosts已被和谐，尝试更换压缩文件内“必看”提供的备用IP或等待更新
2.你被GFW暂时性阻断了，稍后再试
浏览器出现了广告、上网出现了异常
别闹了，快去检查一下你添加的扩展是否有问题（Google 审核门槛很低，导致应用商店充斥着恶意扩展）
如果你觉得hosts文件有问题，删除它即可，对系统无影响
安全软件报风险
hosts其实是个较敏感的文件，它可以把某个域名指定到某个IP上。
例如你要访问淘宝，但他把你劫持到了钓鱼网站，后果可想而知。所以它是各类恶意软件的争夺之地
我们提供的hosts不会做这样缺德的事，所有IP均在Google官方IP段内
我可以在移动设备上使用吗
移动设备上的换行符和我们提供的hosts并不相同，需要自行dos2unix转换
并且我们没有加入 移动页面、Android 服务的支持，故不推荐使用
